# 好物购买系统 - 项目部署运行指南

## 一、环境准备

### 1.1 硬件要求

- CPU：Intel Core i3 及以上
- 内存：16GB 及以上
- 硬盘：至少 10GB 可用空间

### 1.2 软件环境

| 软件      | 版本要求 | 说明          |
| ------- | ---- | ----------- |
| JDK     | 17+  | Java开发环境    |
| Maven   | 3.6+ | 项目构建工具      |
| Node.js | 18+  | 前端运行环境      |
| MySQL   | 8.0+ | 数据库         |
| Navicat | 任意版本 | 数据库管理工具（可选） |

## 二、环境安装配置

### 2.1 安装 JDK 17

#### Windows 系统

1. 下载 JDK 17
   - 访问 Oracle 官网：<https://www.oracle.com/java/technologies/downloads/#java17>
   - 或使用 OpenJDK：<https://adoptium.net/>
   - 选择 Windows x64 版本下载
2. 安装 JDK
   - 双击下载的安装包
   - 按照安装向导完成安装
   - 建议安装路径不要包含中文和空格
3. 配置环境变量
   ```
   右键"此电脑" -> 属性 -> 高级系统设置 -> 环境变量

   新建系统变量：
   - 变量名：JAVA_HOME
   - 变量值：JDK安装路径（如：C:\Program Files\Java\jdk-17）

   编辑 Path 变量，添加：
   - %JAVA_HOME%\bin
   ```
4. 验证安装
   ```powershell
   java -version
   javac -version
   ```
   显示版本信息则安装成功。

### 2.2 安装 Maven

#### Windows 系统

1. 下载 Maven
   - 访问官网：<https://maven.apache.org/download.cgi>
   - 下载 Binary zip archive 文件
2. 解压文件
   - 将下载的 zip 文件解压到指定目录（如：D:\apache-maven-3.9.6）
3. 配置环境变量
   ```
   新建系统变量：
   - 变量名：MAVEN_HOME
   - 变量值：Maven解压路径（如：D:\apache-maven-3.9.6）

   编辑 Path 变量，添加：
   - %MAVEN_HOME%\bin
   ```
4. 验证安装
   ```powershell
   mvn -version
   ```
5. 配置 Maven 镜像（可选，加速下载）

   编辑 `%MAVEN_HOME%\conf\settings.xml`，在 `<mirrors>` 标签内添加：
   ```xml
   <mirror>
       <id>aliyun</id>
       <mirrorOf>central</mirrorOf>
       <name>Aliyun Maven Mirror</name>
       <url>https://maven.aliyun.com/repository/public</url>
   </mirror>
   ```

### 2.3 安装 Node.js

1. 下载 Node.js
   - 访问官网：<https://nodejs.org/>
   - 下载 LTS（长期支持）版本
2. 安装 Node.js
   - 双击安装包，按照向导完成安装
   - 安装时会自动配置环境变量
3. 验证安装
   ```powershell
   node -v
   npm -v
   ```
4. 配置 npm 镜像（可选，加速下载）
   ```powershell
   npm config set registry https://registry.npmmirror.com
   ```

### 2.4 安装配置 MySQL 8

1. 下载 MySQL
   - 访问官网：<https://dev.mysql.com/downloads/mysql/>
   - 选择 Windows 版本下载
2. 安装 MySQL
   - 运行安装程序
   - 选择 "Developer Default" 或 "Server only"
   - 设置 root 用户密码（建议设置为 root，方便记忆）
   - 完成安装
3. 验证安装
   ```powershell
   mysql -u root -p
   ```
   输入密码后能进入 MySQL 命令行则安装成功。

## 三、数据库初始化

### 3.1 使用 Navicat 导入数据库

1. 打开 Navicat，连接到 MySQL 数据库
   - 点击"连接" -> "MySQL"
   - 输入连接名、主机（localhost）、端口（3306）、用户名（root）、密码
   - 点击"测试连接"，成功后点击"确定"
2. 创建数据库并导入数据
   - 右键连接名，选择"新建数据库"
   - 数据库名：`haowu_shop`
   - 字符集：`utf8mb4`
   - 排序规则：`utf8mb4_unicode_ci`
   - 点击"确定"创建数据库
3. 导入 SQL 文件
   - 双击打开 `haowu_shop` 数据库
   - 右键数据库名，选择"运行 SQL 文件"
   - 选择项目中的 `haowu_shop.sql` 文件
   - 点击"开始"执行导入
4. 验证导入
   - 刷新数据库，应能看到以下数据表：
     - t\_user（用户表）
     - t\_merchant（商家表）
     - t\_admin（管理员表）
     - t\_product（商品表）
     - t\_order（订单表）
     - t\_cart（购物车表）
     - t\_category（分类表）
     - t\_coupon（优惠券表）
     - 等其他表

### 3.2 使用命令行导入数据库

```powershell
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE haowu_shop CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 退出 MySQL
exit

# 导入 SQL 文件
mysql -u root -p haowu_shop < haowu_shop.sql
```

## 四、后端项目运行

### 4.1 项目结构

```
backend/
├── src/
│   ├── main/
│   │   ├── java/com/haowu/shop/
│   │   │   ├── config/          # 配置类
│   │   │   ├── controller/      # 控制器层
│   │   │   ├── dto/             # 数据传输对象
│   │   │   ├── entity/          # 实体类
│   │   │   ├── filter/          # 过滤器
│   │   │   ├── mapper/          # 数据访问层
│   │   │   ├── service/         # 业务逻辑层
│   │   │   ├── util/            # 工具类
│   │   │   └── Application.java # 启动类
│   │   └── resources/
│   │       └── application.properties  # 配置文件
│   └── test/
├── uploads/                     # 文件上传目录
└── pom.xml                      # Maven 配置文件
```

### 4.2 修改数据库配置

编辑 `backend/src/main/resources/application.properties` 文件：

```properties
# 数据库配置（根据实际情况修改）
spring.datasource.url=jdbc:mysql://localhost:3306/haowu_shop?useSSL=false&serverTimezone=Asia/Shanghai&createDatabaseIfNotExist=true&useUnicode=true&characterEncoding=utf8
spring.datasource.username=root
spring.datasource.password=root    # 修改为你的 MySQL 密码

# 服务器端口
server.port=8086
```

### 4.3 使用命令行运行后端

```powershell
# 进入后端项目目录
cd backend

# 编译项目（首次运行或代码变更后执行）
mvn clean compile

# 运行项目
mvn spring-boot:run
```

看到以下日志表示启动成功：

```
Started Application in X.XXX seconds
```

### 4.4 使用 IDEA 运行后端

1. 使用 IntelliJ IDEA 打开项目
2. 打开 `backend/src/main/java/com/haowu/shop/Application.java`
3. 点击类名或 main 方法旁边的绿色运行按钮
4. 等待项目启动完成

### 4.5 验证后端运行

访问 <http://localhost:8086/api/products> ，如果返回商品数据 JSON 则后端运行成功。

## 七、常见问题

### 7.1 后端启动失败

**问题：数据库连接失败**

```
解决方案：
1. 确认 MySQL 服务已启动
2. 检查 application.properties 中的数据库配置是否正确
3. 确认数据库 haowu_shop 已创建并导入数据
```

**问题：端口被占用**

```
解决方案：
1. 查看端口占用：netstat -ano | findstr :8086
2. 结束占用进程或修改 application.properties 中的 server.port
```

### 7.2 前端启动失败

**问题：依赖安装失败**

```
解决方案：
1. 清除 npm 缓存：npm cache clean --force
2. 删除 node_modules 目录和 package-lock.json 文件
3. 重新执行 npm install
```

**问题：无法访问后端接口**

```
解决方案：
1. 确认后端已启动并运行在 8086 端口
2. 检查 vite.config.js 中的代理配置是否正确
```

### 7.3 登录失败

**问题：提示密码错误**

```
解决方案：
1. 确认使用的是正确的默认密码：123456
2. 检查数据库中用户数据是否正确导入
```

## 八、生产环境部署（可选）

### 8.1 后端打包

```powershell
cd backend
mvn clean package -DskipTests
```

生成的 jar 包位于 `backend/target/shop-1.0.0.jar`

### 8.2 运行生产包

```powershell
java -jar shop-1.0.0.jar
```

### 8.3 前端打包

```powershell
cd web
npm run build
```

生成的静态文件位于 `web/dist/` 目录，可部署到 Nginx 等 Web 服务器。
