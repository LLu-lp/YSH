# 基于Web的好物购买系统设计与实现

## 一、核心技术栈

### 开发语言

Java

### 前端框架/技术

Vue 3

### 后端框架

Spring Boot

### 数据库

MySQL 8

### 开发工具/环境

1. 开发平台：IntelliJ IDEA 2023.3

2. 操作系统：Windows 10及以上

3. 硬件配置：CPU I3及以上，16G及以上内存

### 二、系统功能模块

系统分为**用户端**、**商家端**、**管理员端**三大模块，各模块功能独立且协同，覆盖购物全流程与平台运营管理全环节，无功能遗漏且补充商家端完整功能。

### （一）用户端

为普通消费用户提供一站式线上购物服务，功能覆盖从身份认证到购物完成的全链路操作，具体如下：

1. 用户注册登录模块：完成身份认证，支持账号登出

2. 首页轮播图模块：展示平台促销、推荐商品等信息

3. 商品浏览模块：自由浏览平台所有商品信息

4. 商品搜索模块：关键词检索目标商品

5. 商品分类筛选模块：按分类精准筛选商品

6. 购物车管理模块：添加商品、调整商品数量等操作

7. 订单创建模块：结算购物车商品，生成订单

8. 支付处理模块：完成订单在线支付（模拟支付）

9. 订单状态跟踪模块：实时查看订单物流、处理进度

10. 订单确认收货模块：商品签收后完成确认收货操作

11. 用户个人信息管理模块：编辑、维护个人基础信息

12. 收货地址管理模块：添加、修改、删除收货地址

13. 收藏夹管理模块：收藏心仪商品，方便后续查看

14. 优惠券管理模块：查看、使用平台发放的优惠券

### （二）商家端

为入驻平台的商家提供店铺运营、商品与订单管理能力，聚焦店铺自主运营与交易处理，补充功能如下：

1. 商家注册登录认证模块：商家账号身份验证，支持登出

2. 店铺信息管理模块：编辑店铺名称、简介、头像等基础信息

3. 商品管理模块

    - 商品信息发布/增加模块：上架新商品，完善商品名称、详情、价格、库存、分类等信息

    - 商品信息修改模块：编辑已上架商品的各类信息

    - 商品信息删除/下架模块：对商品进行下架或彻底删除操作

    - 商品库存管理模块：实时更新商品库存，设置库存预警

4. 订单管理模块

    - 店铺订单查看模块：查看本店所有订单（待付款、待发货、待收货、已完成、售后等状态）

    - 订单状态更新模块：处理订单发货，更新订单物流信息

    - 订单售后处理模块：处理用户的退货、换货、退款等售后申请

5. 店铺数据统计模块：查看本店商品销量、订单量、营业额、用户消费数据等运营指标

6. 店铺优惠券管理模块：自主创建、发放、核销本店专属优惠券

7. 店铺用户管理模块：查看本店的会员、消费用户信息，做简单的用户标签管理

8. 订单历史记录查看模块：查询本店历史所有订单数据，支持筛选与导出

### （三）管理员端

为平台超级管理员提供全局运营管理能力，负责平台整体管控、数据监控与权限管理，具体如下：

1. 管理员登录认证模块：管理员账号专属登录验证，支持登出

2. 数据统计和图表展示模块：平台全局数据可视化，含商品总数、用户总数、商家总数、订单总量、平台营业额等

3. 商品全局管理模块：审核商家上架的商品，对违规商品进行下架、删除操作

4. 商家管理模块：审核商家入驻申请、管理商家信息、冻结/解封商家账号

5. 用户管理模块：查看平台所有用户信息、冻结/解封用户账号、处理用户投诉

6. 订单全局管理模块：查看平台所有订单，协调处理商家与用户的订单纠纷

7. 订单状态全局更新模块：特殊情况下对订单状态进行全局调整

8. 平台优惠券管理模块：创建、发放、管理平台通用优惠券

9. 数据导出功能模块：将平台全局的商品、用户、商家、订单等数据导出为文件

10. 订单历史记录全局查看模块：查询平台所有历史订单数据，支持多条件筛选

11. 系统基础设置模块：维护平台名称、公告、运营规则等基础信息
