# 头像上传和显示问题解决方案

## 问题描述
头像上传成功，返回了 URL（如 `/uploads/avatar/7e13d1c0-cb26-44d4-bf97-0198f0c4fff2.jpg`），但访问时显示 404，图片无法加载。

## 根本原因
1. 上传目录 `uploads/` 不存在或路径不正确
2. Spring Boot 的静态资源映射配置不完善
3. Tomcat 临时目录在重启时被清空

## 解决方案

### 1. 创建上传目录
```bash
# 在后端项目根目录创建 uploads/avatar 目录
mkdir -p uploads/avatar
```

### 2. 更新配置文件

**application.properties**
```properties
# 使用相对路径，指向项目根目录下的 uploads 文件夹
upload.dir=./uploads
```

### 3. 改进 WebConfig 配置

**WebConfig.java**
- 使用 `Paths.get().toAbsolutePath()` 获取绝对路径
- 添加日志输出便于调试
- 配置缓存时间提高性能

### 4. 改进 FileUploadController

**FileUploadController.java**
- 使用 `java.nio.file.Files` 和 `java.nio.file.Paths` 替代 `File` 类
- 使用 `Files.createDirectories()` 确保目录存在
- 使用 `Files.write()` 写入文件

### 5. 前端自动保存

**Profile.vue**
- 上传成功后自动调用 `updateUserInfo` 保存到数据库
- 实时更新头像显示

## 完整流程

1. **用户上传头像**
   ```
   选择图片 → 前端验证 → 上传到 /api/upload/avatar
   ```

2. **后端处理**
   ```
   接收文件 → 验证类型和大小 → 创建目录 → 保存文件 → 返回 URL
   ```

3. **前端保存**
   ```
   获取 URL → 自动调用 updateUserInfo → 保存到数据库 → 更新显示
   ```

4. **访问头像**
   ```
   浏览器请求 /uploads/avatar/xxx.jpg → WebConfig 映射到本地文件 → 返回图片
   ```

## 关键配置

### SecurityConfig.java
```java
// 允许访问上传的文件
.requestMatchers("/uploads/**").permitAll()
// 文件上传需要认证
.requestMatchers("/api/upload/**").authenticated()
```

### WebConfig.java
```java
registry.addResourceHandler("/uploads/**")
        .addResourceLocations("file:" + uploadPath + "/")
        .setCachePeriod(3600);
```

## 测试步骤

1. 重启后端服务
2. 登录用户账号
3. 进入用户信息页面
4. 点击头像上传新图片
5. 确认头像立即显示
6. 刷新页面验证头像已保存

## 常见问题

**Q: 上传成功但图片显示 404**
A: 检查 WebConfig 中的路径映射是否正确，查看后端日志输出的映射路径

**Q: 上传后头像没有更新**
A: 确保前端的 `handleAvatarSuccess` 函数正确调用了 `updateUserInfo`

**Q: 重启后头像消失**
A: 确保 `uploads/` 目录在项目根目录，不要放在 Tomcat 临时目录
